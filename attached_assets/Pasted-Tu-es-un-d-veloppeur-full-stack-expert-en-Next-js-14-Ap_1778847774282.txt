Tu es un développeur full-stack expert en Next.js 14 (App Router) et TypeScript.
Crée un site web complet et professionnel pour une clinique médicale appelée
"Clinique SML". Le site doit être moderne, rassurant et accessible.

─────────────────────────────────────────
STACK TECHNIQUE
─────────────────────────────────────────
- Framework  : Next.js 14 avec App Router
- Langage    : TypeScript strict
- Style      : Tailwind CSS
- Icônes     : lucide-react
- Formulaire : react-hook-form + zod
- Animations : framer-motion

DESIGN ET CHARTE GRAPHIQUE
─────────────────────────────────────────

POLICES (Google Fonts) :
  - Titres (h1, h2, h3, logo) : "Space Grotesk" — weights 500, 600, 700
  - Corps / paragraphes / labels : "Poppins" — weights 300, 400, 500

PALETTE DE COULEURS (nuances de bleu uniquement) :
  --color-bg           : #FFFFFF   ← arrière-plan général (blanc pur)
  --color-text         : #0A0A0A   ← texte principal (noir profond)
  --color-text-muted   : #3D3D3D   ← texte secondaire (gris foncé)
  --color-primary      : #0057B7   ← bleu principal (boutons, CTA, liens actifs)
  --color-primary-dark : #003E8A   ← bleu foncé (hover boutons, titres forts)
  --color-primary-mid  : #1A80D9   ← bleu moyen (icônes, badges, accents)
  --color-primary-soft : #5BABF0   ← bleu doux (bordures, éléments secondaires)
  --color-primary-pale : #C8E6FA   ← bleu très clair (fonds de cartes, tags)
  --color-primary-ghost: #EBF5FD   ← bleu fantôme (sections alternées, inputs)

RÈGLES D'APPLICATION :
  - Fond de page          : #FFFFFF partout
  - Sections alternées    : #EBF5FD (bleu fantôme) pour le contraste doux

  id: string
  name: string
  specialty: string
  imageUrl: string
  available: boolean
  description: string

  id: string
  title: string
  description: string
  icon: string
  duration: string
  price: string

interface AppointmentFormData {
  fullName: string
  email: string
  phone: string
  specialty: string
  date: string
  timePreference: 'morning' | 'afternoon' | 'evening'
  reason: string
}

interface Testimonial {
  id: string
  name: string
  rating: number
  comment: string
  date: string
}
